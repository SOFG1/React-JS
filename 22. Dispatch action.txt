

	На данном этапе у нас есть store и state, также несколько методов интерфейса управления state-ом.

	Но в дальнейшем у нас будет громадный state и большое количество методов управления state-ом.

	У нас уже появились проблемы - много функций в пропсах <App/>.

	Это является одной из причин внедрения dispatch(), который заменит методы управления state-ом(addPost, sendMessage...)









	dispatch(action) - метод, который заменит интерфесные методы store для управления state-ом(sendMessage, assPost...) 

	Т.Е. у нас будет единственный метод, который будет выполняться, а логика будет зависеть от свойств аргумента.

	action - аргумент dispatch, который всегда является обьектом. Принимает тип действия, также другие параметры(текст поста, ид пользователя...)

	action.type = свойсто action-а, который описывает тип действия dispatch().



	PS - dispatch() - метод, который заменяет все интерфейсные методы управления state-ом, который легко прокидывается через props.

	PS - dispatch() тоже нужно байндить к store.		// dispatch.bind(store)

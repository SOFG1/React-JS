






		Action Creator


	На данном этапе у нас есть state в store и методы управления. Главный метод - dispatch(), который принимает аргумент action и делает изменения в state. 

	Но при передаче обьекта action в UI, нужно быть внимательным, чтобы правильно указать свойства и значения для action.

	НО т.к. обьект action создают наши компоненты, нарушается принцип single responsibility. И возрастает ответственность UI компонентов.

	Для этого мы можем делегировать создание обьекта action - Action Creator-у.

	Action Creator - функция, которая создает готовый action, который dispatch-ит UI компенент. 	// Т.е. мы передали ответственность создания обьекта action, от компонента к Action Creator-у.

	


	PS - На данном этапе мы передаем ответственность создания обьекта action от компонентов к BLL. Теперь наш UI берет dispatch и готовый action с бизнеса и вызывает.
